#### Project 1: Enterprise Architecture Decision-Making

**Problem:**

- Architecture review board decision-making was slow and lacked transparency
- Lack of reuse of existing service-oriented architectures (SOA)
- Criteria for vendor selection ambiguous; left to tenured architects who often lacked integration knowledge
- Decisions delayed or rejected without clear justification

**Analysis Performed:**

- Reviewed data on review board decisions and outcomes
- Reviewed API usage data
- Interviewed stakeholders throughout organization

**Solution Developed:**

| Artifact             | Description                                                                                                                          |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| Scoring mechanism    | Assessment criteria across multiple EA views: data, application, development, operations, network, security, integration, compliance |
| Fast-track threshold | Solutions scoring above threshold automatically fast-tracked through review board                                                    |
| SOA metrics          | Tracked both development and consumption of reusable assets                                                                          |

**Outcome:**

- Decision-making accelerated and more robust
- Clear, understood criteria replaced ambiguous interpretations
- Metrics for tracking SOA initiative performance

#### Project 2: Policy Cycle Time (COO-Sponsored)

**Sponsor:** Chief Operating Officer, Allstate Financial

**Problem:**

- SLA for policy processing: 28 days
- Actual processing time: 35 days
- Longer processing â†’ higher rejection rates
- Revenue impact: Policies over $500K particularly susceptible to delays

**Analysis Performed:**

- Analyzed policy cycle time data (duration by department, face value, demographics)
- Value stream analysis of new business process
- Reviewed input/output metrics, issues, roles, and systems at each stage

**Root Cause Discovery:**

| Issue                | Detail                                                                                 |
| -------------------- | -------------------------------------------------------------------------------------- |
| SLA structure        | Each department had SLA as a range with 20% variance (not single number)               |
| Process inefficiency | Departments operating at upper end of their ranges                                     |
| Scorecard illusion   | Individual department scorecards always green, but aggregate always exceeded threshold |

**Solution:**

- Presented findings to COO
- Identified low-hanging fruit to reduce cycle time in each department to average
- COO instructed department leaders to implement changes
- Control plan implemented to monitor improvements

**Outcome:**

- **Policy cycle time reduced:** 35 days â†’ 28 days
- **Policy acceptance rate improved:** 14% increase

### Reason for Leaving

Allstate offered voluntary termination to reduce head office headcount. Was already interviewing at Microsoft, so took the offer and joined Microsoft in April 2006.
